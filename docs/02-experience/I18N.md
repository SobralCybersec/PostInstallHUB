# Internationalization and localization

**N/A.** PostInstallHUB is English-only. No internationalization is planned.

---

## Rationale

- This is a personal tool built by and for Matheus. English is sufficient.
- Terminal scripts have no locale-aware rendering pipeline — there is no i18n framework to plug into.
- The complexity of maintaining translated strings in bash (no gettext setup is planned) is not justified for a solo personal tool.
- All output strings are hard-coded directly in the scripts. This is intentional and acceptable given the single-author, single-language context.

---

## If i18n were ever needed

The correct approach for bash i18n would be GNU gettext (`gettext` / `ngettext` + `.po` files). This is not planned. If it ever becomes necessary, the first step would be extracting all `log_*` strings into a messages file and wrapping them with `gettext`. Record that decision in an ADR at that time.
